# レシメモ 🧾

> レシートをパッと記録して、今月いくら使ったかをすぐ確認できる、シンプルな支出管理アプリです。

## 📱 これは何？

「レシメモ」は、日々の買い物のレシートをもとに支出を手軽に記録できるスマホアプリです。金額・カテゴリ・写真を数タップで入力するだけで、今日・今月の支出とカテゴリ別の内訳がすぐに分かります。

React Native + Expo + TypeScriptで作られた個人開発アプリで、OCRやAIに頼らず「まず手入力でしっかり使えるものを作る」ことを目標にしたMVP（実用最小限の製品）です。

## 💡 解決する課題

- 家計簿アプリは高機能すぎて続かない・入力が面倒で三日坊主になりがち
- レシートを溜め込んでしまい、後でまとめて入力しようとすると挫折する
- 「今月あといくら使えるか」をパッと確認したいだけなのに、アプリを開くたびに手間がかかる

→ レシメモは、**レシートを見ながらその場で数タップ**で記録を終えられることと、**今日・今月の合計をホーム画面で即座に確認できる**ことにこだわりました。

## ✨ 主な機能

| 機能 | 説明 |
|---|---|
| 支出入力 | 金額・日付・カテゴリ・メモを入力して記録 |
| カテゴリ分類 | 食費・交通費など10種類の固定カテゴリから選択 |
| 今日/今月の合計表示 | ホーム画面ですぐに使いすぎをチェック |
| カテゴリ別合計 | 今月何にお金を使っているかが一目で分かる |
| レシート写真の添付 | カメラ撮影 or 写真ライブラリから選択して記録に添付 |
| レシートOCR | 添付したレシート写真から文字を読み取り、テキストとして保存・表示 |
| OCR結果からの自動抽出 | 読み取ったテキストから合計金額・日付・店名をルールベースで抽出し、確認・修正のうえフォームに反映 |
| カテゴリ自動判定 | 店名・メモ・OCRテキストと過去の登録履歴からカテゴリ候補を確信度付きで複数提示し、自動選択（手動変更も可能） |
| グラフ表示 | カテゴリ別支出を円グラフで可視化 |
| 月予算アラート | 月の予算を設定し、80%超で注意、100%超で警告を表示 |
| CSV出力 | 支出データをCSVファイルとして書き出し、共有シートから送信・保存 |
| 支出の検索・絞り込み・並び替え | 店名/メモ/カテゴリ名/OCRテキストで検索し、カテゴリ・期間で絞り込み、日付や金額で並び替え |
| 月別レポート | 今月と前月の比較、カテゴリ別ランキング、一番使ったカテゴリ、支出が一番多かった日などを表示 |
| データのバックアップ・復元 | 支出・予算データをJSONファイルとして書き出し・共有し、別端末や再インストール後に復元できる |
| 設定画面 | CSV出力・バックアップ・復元・予算設定・全データ削除・アプリ情報を1画面に集約 |
| 支出の詳細・編集・削除 | 記録した支出を後から見直し・修正・削除できる |
| ローカル保存 | サーバー不要。すべて端末内（AsyncStorage）に保存 |

## 🛠 使用技術

- **フレームワーク**: Expo SDK 54 / Expo Router v6 / React Native 0.81 / React 19
- **言語**: TypeScript
- **データ保存**: `@react-native-async-storage/async-storage`（ローカルストレージ）
- **写真機能**: `expo-image-picker`（撮影・選択）/ `expo-file-system`（永続保存・CSV/バックアップ書き出し）
- **OCR**: Google Cloud Vision API（レシート写真からのテキスト読み取り）
- **共有・ファイル選択**: `expo-sharing`（CSV・バックアップファイルの共有）/ `expo-document-picker`（復元用ファイルの選択）
- **グラフ**: `react-native-chart-kit` / `react-native-svg`
- **画面遷移**: Expo Router（ファイルベースルーティング）

## 🧠 工夫した点

- **段階的なMVP開発**: 最初からOCRやAI分類、写真添付に手を広げず、「手入力→ローカル保存→グラフ表示」の順に機能を積み上げて、常に動くアプリを保ちながら開発しました。
- **型安全な設計**: `Expense`・`Category`などのデータ型を早い段階で固め、TypeScriptの型チェックを通して画面間のデータの受け渡しミスを防いでいます。
- **壊れにくいローカル保存**: ストレージの読み書きは全て`try/catch`で保護し、失敗時も例外を投げず安全な値（空配列や`null`）を返す設計にして、アプリが不意にクラッシュしないようにしました。
- **写真ファイルの孤立防止**: 選んだ写真は「保存ボタンを押した瞬間」にだけ端末の永続領域へコピーする設計にし、入力を途中でやめても不要なファイルが残らないようにしています。
- **Safe Area対応の一元管理**: `ScreenContainer`という共通コンポーネントを作り、ステータスバーやホームインジケーターとUIが重ならないよう、全画面で同じロジックを使い回せるようにしました。
- **保存フィードバックの分かりやすさ**: 保存成功時は一瞬「✅ 保存しました」と表示し、金額未入力などのエラーはフィールドの直下に理由を添えて表示するなど、初めて使う人でも迷わないよう配慮しました。
- **共通デザインシステム**: 色は`constants/theme.ts`に一元化（通常操作は青緑系、成功はgreen、注意はamber、危険な操作はred系）し、`AppButton`・`Card`・`EmptyState`という共通コンポーネントを作って、ボタン・カード・空データ表示の見た目を全画面で統一しました。背景は暖かみのあるオフホワイトを基調にしています。
- **AI置き換えを見据えたカテゴリ判定の設計**: カテゴリ判定は「過去の同じ店名の履歴 → キーワードルール」の優先順位で確信度付きの候補を返す関数として独立させ、呼び出し側は最初から非同期・複数候補・確信度・判定理由という形を前提にしています。中身をAI APIに差し替えても呼び出し側の変更が不要な構造です。

## 🏗 セットアップ方法

1. 依存パッケージをインストール

   ```bash
   npm install
   ```

2. （Windows環境でTLS証明書エラーが出る場合）

   環境によっては `npm install` や `npx expo start` の実行時に `UNABLE_TO_VERIFY_LEAF_SIGNATURE` のようなTLSエラーが出ることがあります。その場合は以下のように `NODE_OPTIONS` を設定してから実行してください（OSの証明書ストアを利用するようになります）。

   ```powershell
   $env:NODE_OPTIONS="--use-system-ca"; npm install
   ```

3. （OCR機能を使う場合）レシートOCRはGoogle Cloud Vision APIを利用します。

   1. [Google Cloud Console](https://console.cloud.google.com/)でプロジェクトを作成し、Vision APIを有効化してAPIキーを発行します
   2. 悪用防止のため、発行したAPIキーはGoogle Cloud Console側で「Vision APIのみ利用可」に制限することを推奨します
   3. プロジェクトルートに`.env`ファイルを作成し、`.env.example`を参考に以下を設定します

      ```
      EXPO_PUBLIC_GOOGLE_VISION_API_KEY=あなたのAPIキー
      ```

   > ⚠️ `EXPO_PUBLIC_`が付く環境変数はビルド後のアプリ内に平文で埋め込まれます。個人利用のMVPとしては簡易的にこの方式を使っていますが、公開アプリではAPIキーをサーバー経由で扱うなどの対策を検討してください。APIキーを設定しない場合、OCR機能は「設定が完了していません」というメッセージを表示し、他の機能には影響しません。

## ▶️ 起動方法

開発サーバーを起動します。

```bash
npx expo start
```

（TLS証明書エラーが出る場合は `$env:NODE_OPTIONS="--use-system-ca"; npx expo start`）

起動後、ターミナルに表示されるQRコードを **Expo Go** アプリ（iOS/Android）でスキャンするか、`w`キーでWebブラウザ、`a`キーでAndroidエミュレータ、`i`キーでiOSシミュレータを起動できます。

## 📂 プロジェクト構成

- `app/` — 画面（Expo Routerのファイルベースルーティング）
  - `(tabs)/index.tsx` — ホーム画面（合計・予算進捗・カテゴリ別内訳・最近の支出一覧）
  - `add-expense.tsx` — 支出入力画面（モーダル）
  - `expense/[id].tsx` — 支出の詳細画面
  - `edit-expense.tsx` — 支出の編集画面（モーダル）
  - `graph.tsx` — カテゴリ別支出のグラフ画面
  - `set-budget.tsx` — 月予算の設定画面（モーダル）
  - `expense-list.tsx` — 支出の検索・絞り込み・並び替え画面
  - `monthly-report.tsx` — 月別レポート画面
  - `settings.tsx` — 設定画面（CSV出力・バックアップ・復元・予算設定・全データ削除・アプリ情報）
- `components/` — 画面を構成する部品（`AppButton`・`Card`・`EmptyState`などの共通UI、フォーム、カテゴリ選択、写真ピッカー、OCR読み取り、予算進捗、フィルターUI、設定行、Safe Area対応ラッパーなど）
- `services/` — データ永続化・外部連携（`expense-storage.ts`、`receipt-photo-storage.ts`、`budget-storage.ts`、`csv-export.ts`、`ocr-service.ts`、`category-prediction.ts`、`backup-service.ts`）
- `types/` — 支出データの型定義
- `constants/categories.ts` — カテゴリ定義（色・絵文字・ラベル）
- `utils/` — 日付・金額フォーマット・集計・予算判定・CSV変換ロジック
