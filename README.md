# レシメモ

> レシートをパッと記録して、今月いくら使ったかをすぐ確認できる、シンプルな支出管理アプリです。

## 📱 アプリ概要

「レシメモ」は、日々の買い物のレシートをもとに支出を手軽に記録できるスマホアプリです。金額・カテゴリ・写真を数タップで入力するだけで、今日・今月の支出とカテゴリ別の内訳がすぐに分かります。

React Native + Expo + TypeScriptで作られた個人開発アプリです。家計簿アプリにありがちな「多機能すぎて続かない」を避け、レシートを見ながらその場で数タップで記録を終えられることにこだわりました。

## ✨ 主な機能

| 機能 | 説明 |
|---|---|
| 支出記録 | 金額・日付・カテゴリ・メモを入力して記録 |
| レシート写真添付 | カメラ撮影 or 写真ライブラリから選択して記録に添付 |
| カテゴリ分類 | 食費・交通費など固定カテゴリから選択して管理 |
| 今日・今月の支出合計 | ホーム画面ですぐに使いすぎをチェック |
| 最近の支出一覧 | 直近の記録をホーム画面から素早く確認 |
| グラフ表示 | カテゴリ別支出を円グラフで可視化 |
| 月別レポート | 今月と前月の比較、カテゴリ別ランキングなどを表示 |
| 検索・フィルター | 店名/メモ/カテゴリで検索し、期間や金額で絞り込み・並び替え |
| CSV出力 | 支出データをCSVファイルとして書き出し、共有シートから送信・保存 |
| バックアップ・復元 | 支出・予算データをJSONファイルとして書き出し・共有し、別端末や再インストール後に復元 |
| 通知リマインダー | 毎日決まった時刻の記録リマインダー、予算超過通知（ON/OFF・時刻を設定可能） |
| Free / Plus / Pro プラン画面 | 3プランの機能比較を表示 |
| ムダ買い発見🔒 | 今月と先月の支出を自動比較し、増えているカテゴリ・お店に気づける（Pro限定） |
| 節約ミッション🔒 | 支出データから自動生成される今週・今月の節約目標と達成状況を表示（Pro限定） |
| 買い物パターン分析🔒 | 曜日・月初/中/末・カテゴリ・お店ごとの支出の偏りを分析（Pro限定） |

🔒 のついた機能はPro限定機能です（Free/Plusは一部プレビューのみ表示され、プラン画面へ誘導されます）。

## 💎 プラン（Free / Plus / Pro）

| プラン | 主な内容 |
|---|---|
| **Free** | 支出記録・基本グラフ・写真添付・月別合計。Pro限定機能は一部プレビューのみ |
| **Plus** | 支出記録の上限拡大、CSV出力、バックアップ・復元、予算アラート、検索・フィルター、月別レポート |
| **Pro** | Plusの全機能に加え、ムダ買い発見・買い物パターン分析・節約ミッションをフル解放 |

※ Plus/Proは近日提供予定です。

## 🌟 差別化ポイント

- **銀行連携なしで安心**: 口座やクレジットカードには一切接続しません。金融情報を預ける必要がなく、レシートを撮る・入力するだけで使えます。
- **レシート写真で支出を記録**: レシートを撮影するだけで手軽に記録が残せます。
- **ムダ買いに気づける**: 今月と先月の支出を自動比較し、増えているカテゴリ・お店を教えてくれます。
- **節約ミッションで続けやすい**: 支出データから自動生成される無理のない目標で、家計簿を継続しやすくします。
- **シンプルな日本語UI**: 多機能すぎて挫折しがちな家計簿アプリとは逆に、日本語だけのシンプルな画面構成で迷いません。
- **学生・新社会人にも使いやすい**: 家計簿初心者でも直感的に使える、必要最小限の導線にしました。

## 🛠 使用技術

- **フレームワーク**: Expo SDK 54 / Expo Router v6 / React Native 0.81 / React 19
- **言語**: TypeScript
- **データ保存**: `@react-native-async-storage/async-storage`（ローカルストレージ）
- **写真機能**: `expo-image-picker`（撮影・選択）/ `expo-file-system`（永続保存・CSV/バックアップ書き出し）
- **共有・ファイル選択**: `expo-sharing`（CSV・バックアップファイルの共有）/ `expo-document-picker`（復元用ファイルの選択）
- **通知**: `expo-notifications`（毎日のリマインダー・予算超過通知）
- **グラフ**: `react-native-chart-kit` / `react-native-svg`
- **画面遷移**: Expo Router（ファイルベースルーティング）

## 🏗 セットアップ方法

1. 依存パッケージをインストール

   ```bash
   npm install
   ```

2. （Windows環境でTLS証明書エラーが出る場合）

   環境によっては `npm install` や `npx expo start` の実行時に `UNABLE_TO_VERIFY_LEAF_SIGNATURE` のようなTLSエラーが出ることがあります。その場合は以下のように `NODE_OPTIONS` を設定してから実行してください（OSの証明書ストアを利用するようになります）。

   ```powershell
   $env:NODE_OPTIONS="--use-system-ca"; npm install
   ```

## ▶️ 起動方法

開発サーバーを起動します。

```bash
npx expo start
```

（TLS証明書エラーが出る場合は `$env:NODE_OPTIONS="--use-system-ca"; npx expo start`）

起動後、ターミナルに表示されるQRコードを **Expo Go** アプリ（iOS/Android）でスキャンするか、`w`キーでWebブラウザ、`a`キーでAndroidエミュレータ、`i`キーでiOSシミュレータを起動できます。

### 🧑‍💻 開発者向けメモ：課金実装について

Plus/Proの切り替えは現状 `services/billing-service.ts` のスタブ（`purchasePlan()` は常に `{ success: false, reason: 'not-implemented' }` を返す）と、開発ビルド（`__DEV__`）限定の `setCurrentPlanForDemo()` によるローカルデモ切り替えのみです。実際の課金（App Store In-App Purchase）は未接続のため、公開ビルドでは `app/plans.tsx` のPlus/Pro購入ボタンを無効化しています。

- 本命は **App Store In-App Purchase（IAP）**。サブスクの自動更新・解約・返金をOS標準フローに任せられる。
- **RevenueCat** でIAPの実装・レシート検証・契約状態管理を代行させる想定（現状のFree/Plus/Pro 3段階にそのまま乗せやすい）。
- **Apple Payは不採用**。Apple Payはアプリ外で提供される商品向けの決済手段であり、アプリ内デジタル機能（記録上限解放・分析機能など）の課金はApp Storeガイドライン上IAPが必須のため。
- 実装ステップ（案）: ① App Store ConnectでPlus/Proのサブスク商品を作成 → ② RevenueCatプロジェクトを作成し連携 → ③ `purchasePlan()` をRevenueCat SDK呼び出しに置き換え、`getCurrentPlan()` をentitlementsベースに変更 → ④ `plans.tsx` の購入ボタンの無効化を解除 → ⑤ 復元購入（Restore Purchases）を追加。
- アプリアイコン・スプラッシュ・Bundle ID（`com.reshimemo.app`）などApp Store Connect提出に必要な基本設定は `app.json` で管理済み。EASビルド設定（`eas.json`）は整備中。
