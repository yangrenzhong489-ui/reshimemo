# プロジェクト概要

- アプリ名: **レシメモ**
- レシート撮影・支出メモアプリ
- 技術スタック: React Native + Expo + TypeScript（Expo Router使用）

# 開発方針

- **まずMVPを完成させること**を最優先する。MVP機能（支出入力・カテゴリ分類・月別合計・写真添付・グラフ表示）を手入力ベースで完成させるまでは、それ以外の機能を追加しない。
- 以下の機能は後回しにする。実装しない、または着手前に必ずユーザーに確認する。
  - OCR（レシート自動読み取り）
  - AI分類（カテゴリ自動判定）
  - 広告
  - Pro版・課金機能
- UIの文言はすべて**日本語**にする。
- TypeScriptの型を必ず使う。`any`を安易に使わず、データモデル（支出・カテゴリなど）は明確な型/interfaceで定義する。
- コンポーネントは機能ごとに分割する。1ファイルに巨大な画面を書かず、再利用できる単位（入力フォーム、リスト項目、グラフ等）に分ける。
- 画面はシンプルで分かりやすいUIにする。不要な装飾や複雑な導線を避け、迷わず使えることを優先する。
